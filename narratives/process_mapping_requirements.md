**Process Mapping & Requirements Summary for Government Program Teams** 

Mapped end to end workflows and translated stakeholder requirements into concise, readable summaries for cross functional teams. Improved clarity, reduced miscommunication, and supported smoother project execution.

**Project Overview**

This project demonstrates my ability to gather requirements, clarify complex processes, and communicate them in a structured, accessible format for cross functional teams.

**Objective**

To map the workflow of a government program team and create a clear requirements summary that supports system updates, communication alignment, and project planning.

**Approach**

-	Conducted interviews with analysts, program staff, and technical teams.
-	Documented the current state workflow and identified communication gaps.
-	Translated technical requirements into plain language summaries.
-	Created a process map and accompanying documentation for leadership review.

**Outcome**

-	Improved clarity across teams regarding workflow steps and responsibilities.
-	Reduced miscommunication during system updates.
-	Provided a reference document for future process improvements.

**Skills Demonstrated**

Stakeholder communication • Requirements gathering • Process documentation • Cross functional collaboration
